# @engram/client

TypeScript SDK for the ENGRAM MCP memory server.

## Quickstart

```ts
import { EngramClient } from '@engram/client';

const client = new EngramClient({ baseUrl: 'http://localhost:3000', apiKey: 'eng_…' });
await client.ingestConversation({
  userId: 'clm0000000000000000000000',
  turns: [{ role: 'user', content: 'Prefer TypeScript strict mode.' }],
});
const { results } = await client.recall({
  userId: 'clm0000000000000000000000',
  query: 'TypeScript',
});
console.log(results[0]?.memory.content);
```

## API

| Method                      | Description                                                   |
| --------------------------- | ------------------------------------------------------------- |
| `remember(input)`           | Store a single memory (smart create with duplicate detection) |
| `recall(input)`             | Semantic search over long-term memories                       |
| `forget(input)`             | Find and optionally delete memories by concept                |
| `reflect(input)`            | Synthesise structured insights across memories                |
| `promptContext(input)`      | Assemble a token-budgeted, query-ranked context block         |
| `ingestConversation(input)` | Bulk-store conversation turns as individual memories          |
| `close()`                   | Terminate the MCP session                                     |

### Notes

- `userId` must be a CUID or CUID2 string (e.g. generated by `@paralleldrive/cuid2`).
- `apiKey` is forwarded as `Authorization: Bearer <key>` on every HTTP request.
- The session is initialised lazily on the first method call.

## Installation

```bash
pnpm add @engram/client
```
