# WP1 — Marketing Site Validation Report

Site under test: `apps/marketing-site` (Vite + React 18). Deployed artifact: `apps/marketing-site/dist/` in the main checkout (CNAME: `engram.events`).
Date: 2026-07-05. All file:line references are to this worktree (`.claude/worktrees/plans-suite-2026-07`) unless prefixed with `main-checkout:`.

## Executive summary

**26 findings: 3 critical, 7 high, 10 medium, 6 low.** The three criticals: (1) the production domain is broken over HTTPS — `https://engram.events` presents the `*.github.io` certificate (hostname mismatch), so every HTTPS-first browser shows a security interstitial, and HTTP does not redirect (R1/R9); (2) the site's only install instruction, `claude mcp add engram`, is not a valid command (`claude mcp add` requires `<name> <commandOrUrl>`) and nothing installable named `engram` exists — real setup is clone → build → point a client at `node .../dist/main.js` (R2); (3) tied to it, the site omits every real prerequisite (Node/pnpm/Docker/.env/migrations). Highs: the "dream" panel invents a "reconciles contradictions" capability that does not exist in the codebase and misdescribes consolidation; the app-local README describes a CDN/no-build site that hasn't existed since the Vite port; the page is 100% client-rendered with zero meta/OG/favicon/robots; the haze engine forces ~1 layout reflow per animation frame; decorative fake "memories" are exposed to screen readers. Page weight is otherwise healthy: ~184 kB raw / ~59 kB gzip in one JS chunk + inline CSS, no images at all. The claims that matter most — open source (Apache-2.0), MCP-native, the four verbs existing as tools, semantic recall, Redis STM/Postgres LTM — all check out. 13 remediation tasks (R1–R13) follow, sized S/M, mostly parallelizable.

## Method

1. **Information validity** — read every user-visible string in `apps/marketing-site/index.html`, `app.jsx`, `memory-haze.js` (fragment copy), and the app-local `README.md`. Cross-checked each factual claim against: root `CLAUDE.md`, root `README.md`, root `package.json`, `pnpm-workspace.yaml`, `prisma/schema.prisma`, `packages/core/src/mcp/tools/index.ts`, `apps/mcp-server/src/memory/memory.controller.ts` (the real MCP tool registry), `apps/mcp-server/src/memory/decay.service.ts`, `.env.example`, `LICENSE`, `claude_desktop_config.json.example`, and the git remote URL in the main checkout's `.git/config` (read as a file; no git commands were run).
2. **Installation instructions** — extracted the single install affordance shown on the site (`claude mcp add engram` copy-button + GitHub link, `app.jsx:190-213`) and compared against the real setup flows in root `README.md` (three deployment profiles), `CLAUDE.md` dev setup, `claude_desktop_config.json.example`, `docker-compose.yml`, and root `package.json` scripts. Verified each referenced command exists.
3. **Performance** — attempted `pnpm --filter @engram/marketing-site build` from the worktree root (recorded outcome), then built fresh with npm inside the worktree (`apps/marketing-site` has its own `package-lock.json`). Measured per-chunk raw and gzip sizes (`gzip -c | wc -c`). Also inspected the deployed `dist/` in the main checkout READ-ONLY for comparison. Statically analyzed `index.html` (render-blocking resources, meta/SEO), `memory-haze.js` and `smooth-scroll.js` (rAF loops, listener passivity, layout thrash), and `tweaks-panel.jsx`.

External link liveness WAS checked from this environment via curl: `github.com/osirison/engram` → 200; Google Fonts CSS URL → 200; `https://engram.events` → TLS verification failure (wrong certificate — see Performance findings), `http://engram.events` → 200 without HTTPS redirect. Checks NOT performed (recorded as remediation tasks R10, R12): real Lighthouse/Core-Web-Vitals run, runtime FPS/long-task profiling, cross-browser rendering, Google Fonts transfer-size measurement.

## Findings — Performance

### Build reproducibility (measured in this worktree)

- `pnpm --filter @engram/marketing-site build` from repo root → **"No projects matched the filters"**. Cause: the app is deliberately excluded from the workspace (`pnpm-workspace.yaml:3` — `'!apps/marketing-site'`) and carries its own `package-lock.json`. This is intentional and CI-covered (`.github/workflows/node.js.yml:1-7` — its own npm-based verify + GitHub Pages deploy, path-filtered), but the app-local README doesn't say so and CLAUDE.md/root README never mention the site at all. Severity: **low** (works as designed, discoverability gap).
- Fresh build inside the worktree: `npm ci` (clean, 0 vulnerabilities) + `npm run build` → Vite 6.4.3, 29 modules, 521 ms.

### Production availability (measured)

- **CRITICAL:** `https://engram.events` fails TLS verification — the server presents the `CN=*.github.io` certificate (verified via `openssl s_client`; Let's Encrypt cert for github.io, valid 2026-06-04→2026-09-02, hostname mismatch for engram.events). Every modern browser (HTTPS-first) shows a full-page security error. Plain `http://engram.events` returns 200 and does **not** redirect to HTTPS. Likely cause: GitHub Pages custom-domain cert not provisioned / "Enforce HTTPS" not enabled.
